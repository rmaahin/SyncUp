## Key Features (All 11 Required)
0. **Student onboarding & availability management**: Students receive an invite link, land on `/student/onboarding`, fill out a structured form (skills with proficiency ratings, hours/week, preferred working times, blackout periods like exam weeks, timezone). They OAuth-link Google (Calendar + Docs), GitHub, and Trello in one flow. Profiles are stored in state. Mid-project, students can update availability via `/student/settings`. If availability drops by ≥30% or a new blackout conflicts with assigned deadlines, the Supervisor triggers the Delegation agent to re-optimize affected tasks only (not a full re-plan). All changes logged to `availability_updates` in state for audit.
1. **Skill-based task assignment**: Delegation agent matches tasks to students by declared skills and expertise levels
2. **Realistic deadline pacing**: Pacing algorithm in `services/pacing.py` distributes deadlines proportionally across the project timeline using a burn-down curve. Never front-loads or crams. Takes final deadline, task effort estimates, and student availability (hours/week) as inputs.
3. **Meeting orchestration**: Meeting Coordinator agent finds optimal overlapping availability, schedules recurring meetings every X days via Google Calendar MCP, generates pre-meeting agendas from project state, ingests meeting notes from shared Google Doc afterward
4. **Trello integration**: Publishing agent creates a Trello board with To Do / In Progress / Review / Done lists. Each task = a Trello card with assignee, due date, urgency label, dependency checklist
5. **Google Calendar reminders**: Deadline events created per student per task with tiered reminders (48h, 24h, 2h before deadline)
6. **Continuous progress monitoring**: Event-driven Progress Tracking agent logs every contribution to the immutable contribution ledger in LangGraph state
7. **Activity monitoring**: GitHub webhooks for commits/PRs, Google Docs edit tracking, Trello card movements — all feed into the tracking agent
8. **Dual dashboards**: Student dashboard (personal tasks, team progress, meetings, nudges) and Professor dashboard (per-team overview, per-student contribution metrics, intervention history, audit trail, peer review aggregation)
9. **Peer rating form**: Peer Review Generator creates a structured Likert-scale (1-5) form at project close. Rates: contribution quality, communication, reliability, collaboration, technical competency. Anonymous. Aggregated with bias detection.
10. **Urgency tags**: Every task gets an urgency level — Critical (on blocking path), High (near-deadline), Medium (standard), Low (flexible). Stored in task model, displayed as Trello labels, factored into conflict resolution severity.
